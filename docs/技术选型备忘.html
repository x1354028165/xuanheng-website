技术选型备忘

本文档记录官网重构项目的技术实现选型，与 PRD 分开维护。
PRD 描述"要做什么"，本文档记录"怎么做"。
状态：⬜ 待决策 / ✅ 已确认 / 🔄 待评估

一、前端框架

候选优势劣势状态
Next.js（React）生态最强，ISR 缓存策略适合新闻/品牌页毫秒级访问，SSR + SEO 成熟，国内外 CDN 分离方案案例丰富团队需有 React 基础⬜ 待确认
Nuxt.js（Vue）若现有云平台前端大量使用 Vue，团队认知切换成本低部分企业级方案不如 Next.js 成熟⬜ 待确认

决策关键点： 确认现有开发团队（AU-03 项目等）主要使用 React 还是 Vue。

二、CMS 框架

候选优势劣势状态
Strapi（Node.js）自带 RBAC 权限管理、文件资产管理，2-4 周内可完成核心业务逻辑配置，社区活跃需后端有 Node.js 基础；复杂自定义逻辑需二次开发🔄 推荐评估
Directus（Node.js）数据模型完全自定义，适合 BMS 告警字典等复杂结构，自带 REST + GraphQL API上手曲线稍陡🔄 推荐评估
完全自研最灵活，无外部依赖工期 3–6 个月，当前阶段过重❌ 不推荐
SaaS CMS（Contentful / Sanity）上手快多语言字典管理等自定义逻辑受限，长期按量计费❌ 不推荐

决策关键点： 确认后端团队 Node.js 熟练程度。Strapi 适合快速上线，Directus 适合数据模型复杂场景。

候选状态

四、CDN 服务商

需求（来自 PRD §7.11）：

  • 国内用户走国内加速节点
  • 海外用户走境外加速节点
  • 支持按地理位置智能路由
  • ICP 备案在国内节点完成后方可上线
候选适用场景状态
阿里云 CDN + 阿里云海外节点国内覆盖好，海外节点一般⬜ 待评估
腾讯云 CDN与阿里云类似⬜ 待评估
阿里云（国内）+ Cloudflare（海外）国内外分别最优，但需双平台维护⬜ 待评估
单一全球 CDN（Cloudflare）管理简单，但国内速度受限，ICP 备案需额外配置❌ 不推荐（ICP 合规风险）

备注： 对象存储（OSS 等）选型随 CDN 服务商确定，建议同一厂商减少跨平台传输成本。

五、机器翻译 API

需求（来自 PRD §7.3）：

  • 以 en-US 为源，生成 de/fr/pt/es/ru 五种语言初稿
  • 人工校对后发布
  • CMS 后台一键触发翻译
候选质量价格状态

Google Cloud Translation覆盖语种最广按字符计费⬜ 备选
Azure Cognitive Services质量一般，价格低按字符计费⬜ 备选

六、API 文档渲染（开发者中心，后续 P1）

开发者中心本期暂缓，以下为后续启动时的技术方案参考。
  • 推荐方案： OpenAPI Spec（YAML）描述 API，使用 Redoc 或 Swagger UI 自动渲染
  • 后端维护： 后端维护 .yaml spec 文件，前端不需要手动写文档页
  • 权限控制： 在 API 网关层实现，无权限的端点返回 403，不在前端做伪隐藏

七、文件下载安全

需求（来自 PRD §8）：

  • 权限受控文件不能直接暴露 CDN 链接
  • 防止链接外泄后被无限次使用

推荐方案： 对象存储签名临时链接（Pre-signed URL），有效期 5 分钟

  • 阿里云 OSS / 腾讯云 COS 均支持此功能
  • 用户点击下载时，后端实时生成签名链接并返回，不存储静态链接

八、SSO 接口（待对接）

需求（来自 PRD §8）：

  • 官网客户端登录复用爱尚管理后台账号体系
  • 登录后携带 token，官网服务端校验获取用户标签

待确认（需与爱尚团队对接）：

  • [ ] 用户标签（产品线权限）读取方式：JWT Claims 还是单独 API？
  • [ ] 接口地址 / 文档
