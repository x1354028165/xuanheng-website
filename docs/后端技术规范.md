## 二、API 设计规范

### 基础规则
- RESTful 风格，JSON 格式
- 所有接口添加 `/api` 前缀
- 版本化：`/api/v1/...`
- 使用 HTTPS，HTTP 强制跳转

### URL 命名
```
GET    /api/v1/products              # 列表
GET    /api/v1/products/:slug        # 详情
POST   /api/v1/leads                 # 创建线索
PATCH  /api/v1/leads/:id             # 更新状态
DELETE /api/v1/brands/:id            # 删除（软删除）
```

### 响应格式
```json
// 成功响应
{
  "data": { ... } | [ ... ],
  "meta": {
    "pagination": { "page": 1, "pageSize": 20, "total": 100 }
  }
}

// 错误响应
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "邮箱格式不正确",
    "details": { "field": "email" }
  }
}
```

### 多语言参数
```
GET /api/v1/products?locale=zh-CN
GET /api/v1/solutions?locale=en-US&populate=*
```

## 三、数据库 Schema 核心模型

### products（产品）
```sql
id          UUID PRIMARY KEY DEFAULT gen_random_uuid()
slug        VARCHAR(100) UNIQUE NOT NULL   -- URL slug，如 neuron-ii
type        VARCHAR(20) NOT NULL           -- 'hardware' | 'cloud'
status      VARCHAR(20) DEFAULT 'published' -- 'draft' | 'published' | 'archived'
sort_order  INTEGER DEFAULT 0
created_at  TIMESTAMP DEFAULT NOW()
updated_at  TIMESTAMP DEFAULT NOW()

-- 多语言内容（单独表）
-- product_translations (product_id, locale, name, tagline, description, ...)
```

### product_translations（产品多语言内容）
```sql
id          UUID PRIMARY KEY
product_id  UUID REFERENCES products(id)
locale      VARCHAR(10) NOT NULL           -- zh-CN, en-US, zh-TW ...
name        VARCHAR(200) NOT NULL
tagline     VARCHAR(500)                   -- 定位一句话
description TEXT                          -- 富文本 HTML
translated_at TIMESTAMP
translated_by VARCHAR(100)               -- 'human' | 'machine'
status      VARCHAR(20) DEFAULT 'draft'   -- 'draft' | 'published'
UNIQUE(product_id, locale)
```

### solutions（解决方案场景）
```sql
id, slug, status, sort_order, created_at, updated_at
-- solution_translations: (solution_id, locale, name, tagline, background, description, highlights_json, ...)
```

### product_solution_links（产品-解决方案双向关联）
```sql
id            UUID PRIMARY KEY
product_id    UUID REFERENCES products(id)
solution_id   UUID REFERENCES solutions(id)
show_on_product  BOOLEAN DEFAULT TRUE      -- 在产品页显示场景跳转
show_on_solution BOOLEAN DEFAULT TRUE      -- 在解决方案页显示产品跳转
sort_order    INTEGER DEFAULT 0
UNIQUE(product_id, solution_id)
```

### files（文件资产）
```sql
id, name, version, type (doc|software|firmware|other),
platform (win|mac|ios|android|universal),
oss_key, cdn_url, file_size, download_count,
status (published|archived), sort_order,
related_product_ids JSON,                 -- 关联产品 UUID 数组
created_at, updated_at
```

### brands（兼容品牌）
```sql
id, name, logo_url, logo_oss_key,
connection_types JSON,                    -- ['cloud', 'local']
protocols JSON,                           -- ['Modbus', 'OCPP']
capabilities JSON,                        -- ['telemetry', 'control']
status (supported|developing|planned),
show_on_homepage BOOLEAN DEFAULT FALSE,
sort_order, created_at, updated_at
-- brand_translations: (brand_id, locale, description)
```

### leads（线索/表单提交）
```sql
id, type (inquiry|repair|api_request),
name, company, email, phone,
intent_category VARCHAR(100),            -- 商务咨询意向分类
content TEXT,
source_page VARCHAR(500),               -- 来源 URL
status (pending|following|closed),
assigned_to VARCHAR(100),
notified_at TIMESTAMP,                  -- 企微/邮件通知时间
user_email_sent_at TIMESTAMP,           -- 用户回执发送时间
created_at, updated_at
```

### i18n_messages（多语言词条）
```sql
id, key VARCHAR(200) UNIQUE NOT NULL,    -- 如 nav.solutions
scope VARCHAR(100),                      -- 词条分组
created_at, updated_at

-- i18n_message_translations: (message_id, locale, value, status)
```

### notification_configs（通知配置）
```sql
id, type (inquiry|repair|api_request),
wechat_webhook_url TEXT,
internal_emails JSON,                    -- 收件邮箱数组
email_reply_template_id UUID,           -- 关联邮件模板
enabled BOOLEAN DEFAULT TRUE,
updated_at
```

## 四、文件上传规范

### 上传流程
```
前端 → POST /api/v1/media/presign → 获取 OSS 预签名上传 URL
前端 → PUT {presign_url} → 直传 OSS（不经过 CMS 服务器）
前端 → POST /api/v1/media/confirm → 通知 CMS 记录文件元数据
CMS  → 自动生成 CDN 访问链接存入数据库
```

### 限制
| 文件类型 | 最大大小 | 允许格式 |
|------|------|------|
| 图片 | 10MB | jpg / png / webp / svg |
| 文档 | 100MB | pdf / docx / xlsx / zip |
| 固件 | 500MB | bin / hex / zip |
| 软件安装包 | 500MB | exe / dmg / apk / zip |

### 图片处理
- 上传后自动生成 WebP 版本
- 生成三种尺寸：original / medium(800px) / thumbnail(200px)
- 记录所有尺寸的 CDN URL

## 五、邮件发送规范

```javascript
// 表单提交后触发（异步队列，不阻塞响应）
await emailQueue.add('send-notification', {
  to: config.internal_emails,
  template: 'lead-notification',
  data: { lead, sourceUrl }
})

await emailQueue.add('send-user-receipt', {
  to: lead.email,
  template: `user-receipt-${lead.type}`,
  locale: lead.locale,
  data: { lead }
})
```

**规则：**
- 邮件发送必须异步处理（队列），不在请求响应链中同步发送
- 发送失败重试 3 次，间隔指数退避
- 记录发送状态到 `leads.user_email_sent_at`
- 敏感信息（SMTP 密码）只通过环境变量注入

## 六、安全规范

| 类型 | 要求 |
|------|------|
| CORS | 只允许官网域名（alwayscontrol.com.cn / staging 域名） |
| 速率限制 | 表单提交接口：同 IP 每分钟最多 5 次 |
| 输入校验 | 所有输入字段服务端校验，不依赖前端校验 |
| XSS | 富文本内容存储前经 DOMPurify 过滤 |
| SQL 注入 | 使用 ORM（TypeORM / Knex），不拼接 SQL 字符串 |
| 文件类型 | 上传前检查 Magic Bytes，不只信任扩展名 |
| 敏感日志 | 日志中不输出邮件地址、手机号等 PII |
| HTTPS | 所有接口强制 HTTPS，HSTS 头部 |

## 八、部署要求

- Node.js 进程由 PM2 守护，崩溃自动重启
- 数据库连接池大小：生产 ≥ 10，Staging ≥ 3
- 日志输出到文件，保留 30 天，ERROR 级别触发告警
- 生产数据库每日自动备份，保留 7 天
- Staging 数据库可一键从生产快照恢复（用于测试数据同步）
