三、多语言实现规范

URL 结构


/zh-CN/           首页（简体中文）
/en-US/           首页（英文）
/zh-TW/           首页（繁体，自动生成）
/zh-CN/solutions/hems
/en-US/products/neuron-ii

语言配置

  • 语言列表从 CMS「语言管理」API 动态拉取，不硬编码
  • 默认语言检测顺序：URL 路径 → 用户偏好（localStorage） → IP 地理位置 → zh-CN
  • 切换语言时保持当前页面路径，仅替换 locale 前缀

i18n 实现

  • UI 固定词条（按钮/导航/错误提示）通过 CMS 多语言字典 API 获取
  • 页面内容（产品描述/解决方案文案）通过各内容 API 直接返回对应语言版本
  • 未翻译内容按回退链处理：zh-TW → zh-CN；de/fr/pt/es/ru → en-US → zh-CN

每页必须包含

六、组件规范

命名规则

  • 组件文件：PascalCase.tsx（如 ProductCard.tsx）
  • 工具函数：camelCase.ts（如 formatDate.ts）
  • CSS Module：ComponentName.module.css
  • 常量：UPPER_SNAKE_CASE

Props 规范


// 每个组件必须定义 Props 类型
interface ProductCardProps {
  product: Product        // 明确类型，不用 any
  locale: SupportedLocale // 多语言必传
  className?: string      // 可选样式扩展
}

禁止事项

  • ❌ 不使用 any 类型
  • ❌ 不在组件内直接调用 CMS API（通过 lib/cms 封装）
  • ❌ 不使用 document / window（SSR 不可用）
  • ❌ 不内联样式（style={{ color: 'red' }}）

七、CMS 数据获取


// lib/cms/index.ts 统一封装
export async function getProducts(locale: string): Promise<Product[]> {
  const res = await fetch(`${CMS_API}/products?locale=${locale}`, {
    next: { revalidate: 300 }, // ISR：5分钟缓存
  })
  if (!res.ok) throw new Error('Failed to fetch products')
  return res.json()
}

// 静态生成页面（产品/解决方案）
export async function generateStaticParams() {
  const products = await getProducts('zh-CN')
  return products.map(p => ({ slug: p.slug }))
}

九、环境变量


# .env.local（本地，不提交 git）
NEXT_PUBLIC_CMS_API_URL=http://localhost:1337/api
NEXT_PUBLIC_SITE_URL=http://localhost:3000

# .env.production（生产）
NEXT_PUBLIC_CMS_API_URL=https://cms.alwayscontrol.com.cn/api
NEXT_PUBLIC_SITE_URL=https://alwayscontrol.com.cn

规则：

  • 含有 NEXT_PUBLIC_ 前缀的变量会暴露到客户端，不放敏感信息
  • 真实密钥（邮件/企微/翻译API）只放服务端环境变量，不加 NEXT_PUBLIC_ 前缀
  • 敏感变量不提交 git，通过部署平台（CI/CD）注入
