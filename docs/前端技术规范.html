SECTION 01

一、技术选型决策

1.1 前端框架：Next.js vs Nuxt.js

对比维度Next.js 14+ (React)Nuxt.js 3 (Vue)
SSR / SSG / ISR 支持✅ 三者原生支持，ISR 成熟稳定，revalidate 粒度到页面级✅ SSR/SSG 支持良好，ISR 通过 routeRules 配置，成熟度稍逊
多语言路由（hreflang）✅ next-intl 生态成熟，[locale] 动态路由 + generateMetadata 原生支持 hreflang✅ @nuxtjs/i18n 可用，但配置复杂度较高
CMS 集成生态✅ Strapi/Contentful/Sanity 官方均提供 Next.js Starter，文档完善⚠️ 集成方案存在但官方模板较少
Claude AI 代码生成质量✅ React/Next.js 训练数据量极大，生成代码质量高、一致性好⚠️ Vue/Nuxt 训练数据量较少，生成代码偶需手动修正
React Server Components✅ 原生支持 RSC，默认零 JS bundle，SEO 友好❌ 不支持（Vue 无此概念）
生态规模（npm 周下载量）✅ ~600 万/周，React 生态 Top 1 元框架⚠️ ~50 万/周，Vue 生态主力但规模差距明显
部署平台原生支持✅ Vercel 原生支持（同公司产品），零配置部署⚠️ 需额外配置 Nitro 预设
选型结论：Next.js 14+（App Router）。Next.js 在 ISR 成熟度、React Server Components、CMS 集成生态以及 AI 辅助开发质量方面均具有显著优势。作为 Vercel 的第一方产品，部署链路零摩擦。React 生态的绝对规模意味着更丰富的组件库、更快的问题排查和更稳定的长期维护。对于本项目以 SEO 为核心、以 CMS 驱动内容、以 Claude 辅助开发的技术约束，Next.js 是唯一合理选择。

1.2 CMS 方案：Strapi v4 vs Directus

对比维度Strapi v4Directus
内容建模可视化✅ Content-Type Builder GUI，非技术人员可直接操作，字段拖拽创建✅ 可视化建模，但基于数据库表映射，概念门槛稍高
原生多语言字段✅ i18n 插件原生支持，每个 Content Type 可独立启用多语言⚠️ 通过 Translations M2M 关系实现，配置复杂度较高
REST + GraphQL 双协议✅ REST 默认启用，GraphQL 通过官方插件一键启用✅ REST + GraphQL 均原生支持
媒体库管理✅ 内置媒体库，支持图片裁剪/格式转换/响应式断点✅ 内置 DAM，功能完善
Webhook 支持（ISR 触发）✅ 内置 Webhook，支持 entry.create/update/delete 等粒度事件✅ 支持 Webhook，事件类型丰富
社区规模（GitHub Stars）✅ 65k+ Stars，npm 插件生态丰富⚠️ 29k+ Stars，社区活跃但规模较小
自托管成本✅ Node.js + SQLite/PostgreSQL，轻量可控✅ Node.js + 任意 SQL 数据库
选型结论：Strapi v4。Strapi 的 i18n 原生支持直接满足本项目 8 语言需求，无需额外建模。其 Content-Type Builder 的可视化程度更高，适合非技术运营人员独立维护内容。65k+ Stars 的社区规模意味着更完善的文档和更快的问题解决。Webhook 内置支持使得 ISR 触发链路开箱即用，与 Next.js 的 revalidate 机制无缝对接。

1.3 样式方案：Tailwind CSS

选择理由：

  • 原子化 CSS：无需命名 CSS 类，直接在 JSX 中用 utility class 描述样式，消除样式文件维护成本
  • 与 Next.js 无缝集成：create-next-app 官方模板默认集成 Tailwind CSS，零配置启动
  • JIT 模式按需生成：仅生成实际使用的 CSS，生产构建体积极小（通常 < 10KB gzip）
  • 适合 AI 代码生成：Tailwind 的声明式语法与 Claude 的代码生成模式高度契合，减少样式相关的生成错误
  • 设计系统一致性：通过 tailwind.config.js 统一定义颜色/字体/间距，确保全站视觉一致

1.4 部署平台：Vercel

选择理由：

  • Next.js 原生部署平台：Vercel 是 Next.js 的开发商，ISR / Middleware / Edge Functions 等高级特性在 Vercel 上获得第一时间支持和最佳性能
  • 全球 Edge Network CDN：静态资源和 ISR 页面通过全球边缘节点分发，海外用户访问延迟极低
  • Preview 部署：每个 Git 分支自动生成独立预览 URL，支持设计评审和 QA 验收
  • 环境变量管理：Production / Preview / Development 三套环境隔离，敏感变量安全存储
  • 零配置 CI/CD：推送 Git 即触发构建部署，无需维护额外的 CI 管道

3.2 渲染模式说明

模式全称适用场景说明
ISRIncremental Static Regeneration内容 CMS 驱动、需频繁更新的页面构建时生成静态页面，过期后下次请求触发后台重新生成。兼顾 SEO 与内容实时性。默认 revalidate: 3600（1 小时），可通过 Webhook 即时刷新。
SSGStatic Site Generation内容极少变化的页面构建时一次性生成，不设 revalidate。变更需重新部署。
SSRServer-Side Rendering本项目暂无此需求每次请求都在服务端渲染。TTFB 较高，仅在需要请求时实时数据时使用。

3.3 动态路由与 generateStaticParams

所有动态路由（[slug] / [scenario]）必须实现 generateStaticParams，确保构建时预生成所有已知路径：

// app/[locale]/products/[slug]/page.tsx
export async function generateStaticParams() {
  const products = await fetchStrapi<StrapiResponse<Product[]>>(
    '/products?fields[0]=slug'
  );
  const locales = ['zh-CN', 'en-US', 'zh-TW', 'de', 'fr', 'pt', 'es', 'ru'];

  return locales.flatMap(locale =>
    products.data.map(p => ({
      locale,
      slug: p.attributes.slug,
    }))
  );
}

3.4 路由命名规范

  • 路径段使用 小写英文 + 连字符（kebab-case）：/about/news，禁止驼峰或下划线
  • 动态参数使用 语义化名称：[slug]（通用）、[scenario]（场景）、[locale]（语言）
  • 禁止使用数字 ID 作为 URL 路径（如 /products/123），必须使用 slug

4.4 降级策略

核心原则：宁可显示降级内容，也不允许白屏。 
// lib/i18n.ts
import { notFound } from 'next/navigation';

const SUPPORTED_LOCALES = ['zh-CN', 'en-US', 'zh-TW', 'de', 'fr', 'pt', 'es', 'ru'] as const;
type SupportedLocale = typeof SUPPORTED_LOCALES[number];

export function isValidLocale(locale: string): locale is SupportedLocale {
  return SUPPORTED_LOCALES.includes(locale as SupportedLocale);
}

export async function getMessages(locale: string) {
  // 验证 locale 合法性
  if (!isValidLocale(locale)) {
    notFound();
  }

  try {
    return (await import(`../messages/${locale}.json`)).default;
  } catch {
    // 降级链：任意语言 → zh-CN，绝不抛错
    console.warn(`[i18n] Missing messages for locale: ${locale}, falling back to zh-CN`);
    return (await import('../messages/zh-CN.json')).default;
  }
}

// CMS 内容降级：Strapi 多语言字段为空时的降级
export function getLocalizedField<T>(
  data: Record<string, T>,
  locale: string,
  fallbackLocale: string = 'zh-CN'
): T | undefined {
  return data[locale] ?? data[fallbackLocale] ?? Object.values(data)[0];
}

5.1 每页必须包含的元数据

每个页面文件必须导出 generateMetadata 函数，包含以下必选字段：

// app/[locale]/page.tsx
import { Metadata } from 'next';

interface PageProps {
  params: { locale: string };
}

export async function generateMetadata({ params }: PageProps): Promise<Metadata> {
  const { locale } = params;
  const t = await getMessages(locale);
  const siteUrl = process.env.NEXT_PUBLIC_SITE_URL || 'https://alwayscontrol.com.cn';
  const isProduction = process.env.NEXT_PUBLIC_ENV === 'production';

  return {
    title: `${t.home.hero.title} | AlwaysControl`,
    description: t.home.meta.description,  // 100-160 字符

    // hreflang 多语言替代链接
    alternates: {
      canonical: `${siteUrl}/${locale}`,
      languages: {
        'zh-CN': `${siteUrl}/zh-CN`,
        'en-US': `${siteUrl}/en-US`,
        'zh-TW': `${siteUrl}/zh-TW`,
        'de': `${siteUrl}/de`,
        'fr': `${siteUrl}/fr`,
        'pt': `${siteUrl}/pt`,
        'es': `${siteUrl}/es`,
        'ru': `${siteUrl}/ru`,
        'x-default': `${siteUrl}/en-US`,
      },
    },

    // Open Graph
    openGraph: {
      title: `${t.home.hero.title} | AlwaysControl`,
      description: t.home.meta.description,
      url: `${siteUrl}/${locale}`,
      siteName: 'AlwaysControl',
      images: [{
        url: `${siteUrl}/images/og-default.jpg`,
        width: 1200,
        height: 630,
        alt: 'AlwaysControl Energy Management',
      }],
      locale: locale,
      type: 'website',
    },

    // Robots：Staging 环境强制 noindex
    robots: isProduction ? 'index,follow' : 'noindex,nofollow',
  };
}

5.2 hreflang 实现

Next.js 14 的 alternates.languages 会自动在 <head> 中生成 <link rel="alternate" hreflang="..."> 标签。关键规则：

  • 每个页面必须声明所有 8 种语言 + x-default（指向 en-US）
  • canonical 指向当前语言版本的绝对 URL
  • 动态页面（产品/新闻）的 hreflang URL 必须包含对应的 slug

5.3 Sitemap 自动生成

// app/sitemap.ts
import { MetadataRoute } from 'next';
import { fetchStrapi } from '@/lib/strapi';

const SITE_URL = process.env.NEXT_PUBLIC_SITE_URL || 'https://alwayscontrol.com.cn';
const LOCALES = ['zh-CN', 'en-US', 'zh-TW', 'de', 'fr', 'pt', 'es', 'ru'];

export default async function sitemap(): Promise<MetadataRoute.Sitemap> {
  // 静态页面
  const staticPages = ['', '/products', '/solutions', '/ecosystem',
    '/support', '/about', '/about/careers', '/contact', '/privacy'];

  const staticEntries = staticPages.flatMap(page =>
    LOCALES.map(locale => ({
      url: `${SITE_URL}/${locale}${page}`,
      lastModified: new Date(),
      changeFrequency: 'weekly' as const,
      priority: page === '' ? 1.0 : 0.8,
      alternates: {
        languages: Object.fromEntries(
          LOCALES.map(l => [l, `${SITE_URL}/${l}${page}`])
        ),
      },
    }))
  );

  // 动态页面：产品
  const products = await fetchStrapi<any>('/products?fields[0]=slug&fields[1]=updatedAt');
  const productEntries = products.data.flatMap((p: any) =>
    LOCALES.map(locale => ({
      url: `${SITE_URL}/${locale}/products/${p.attributes.slug}`,
      lastModified: new Date(p.attributes.updatedAt),
      changeFrequency: 'weekly' as const,
      priority: 0.7,
    }))
  );

  // 动态页面：新闻
  const news = await fetchStrapi<any>('/articles?fields[0]=slug&fields[1]=updatedAt');
  const newsEntries = news.data.flatMap((n: any) =>
    LOCALES.map(locale => ({
      url: `${SITE_URL}/${locale}/about/news/${n.attributes.slug}`,
      lastModified: new Date(n.attributes.updatedAt),
      changeFrequency: 'monthly' as const,
      priority: 0.5,
    }))
  );

  return [...staticEntries, ...productEntries, ...newsEntries];
}

7.2 ISR 触发机制（Strapi Webhook → Next.js revalidate）

Strapi 后台发布/更新内容 ↓ Strapi Webhook POST → https://your-site.vercel.app/api/revalidate ↓ (携带 secret + 变更的 content type + entry id) Next.js API Route 验证 secret → 调用 revalidatePath() / revalidateTag() ↓ 下次用户访问该页面时，Vercel 在后台生成新版本静态页面 ↓ 新页面即时生效（用户无感知）

Webhook 端点实现

// app/api/revalidate/route.ts
import { revalidatePath } from 'next/cache';
import { NextRequest, NextResponse } from 'next/server';

export async function POST(request: NextRequest) {
  const secret = request.nextUrl.searchParams.get('secret');

  // 验证 Webhook 密钥
  if (secret !== process.env.REVALIDATE_SECRET) {
    return NextResponse.json({ message: 'Invalid secret' }, { status: 401 });
  }

  try {
    const body = await request.json();
    const { model, entry } = body;

    // 根据内容类型刷新对应页面
    const locales = ['zh-CN', 'en-US', 'zh-TW', 'de', 'fr', 'pt', 'es', 'ru'];

    switch (model) {
      case 'product':
        locales.forEach(locale => {
          revalidatePath(`/${locale}/products`);
          if (entry?.slug) {
            revalidatePath(`/${locale}/products/${entry.slug}`);
          }
        });
        break;
      case 'article':
        locales.forEach(locale => {
          revalidatePath(`/${locale}/about/news`);
          if (entry?.slug) {
            revalidatePath(`/${locale}/about/news/${entry.slug}`);
          }
        });
        break;
      case 'solution':
        locales.forEach(locale => {
          revalidatePath(`/${locale}/solutions`);
        });
        break;
      case 'home-page':
        locales.forEach(locale => revalidatePath(`/${locale}`));
        break;
      default:
        // 未知类型：刷新所有页面
        revalidatePath('/', 'layout');
    }

    return NextResponse.json({ revalidated: true, model });
  } catch (err) {
    return NextResponse.json({ message: 'Error revalidating' }, { status: 500 });
  }
}

7.3 Strapi API 客户端封装

// lib/strapi.ts
const STRAPI_URL = process.env.STRAPI_API_URL;
const STRAPI_TOKEN = process.env.STRAPI_API_TOKEN;

if (!STRAPI_URL) throw new Error('Missing STRAPI_API_URL env variable');
if (!STRAPI_TOKEN) throw new Error('Missing STRAPI_API_TOKEN env variable');

interface StrapiResponse<T> {
  data: T;
  meta: {
    pagination?: {
      page: number;
      pageSize: number;
      pageCount: number;
      total: number;
    };
  };
}

export async function fetchStrapi<T>(
  path: string,
  options?: RequestInit & { revalidate?: number }
): Promise<StrapiResponse<T>> {
  const { revalidate = 3600, ...fetchOptions } = options ?? {};

  const res = await fetch(`${STRAPI_URL}/api${path}`, {
    headers: {
      Authorization: `Bearer ${STRAPI_TOKEN}`,
      'Content-Type': 'application/json',
    },
    next: { revalidate },  // ISR 缓存时间
    ...fetchOptions,
  });

  if (!res.ok) {
    throw new Error(
      `Strapi fetch failed: ${res.status} ${res.statusText} — path: ${path}`
    );
  }

  return res.json();
}

// lib/strapi.ts — 增加 safeFetch 封装
export async function safeFetchStrapi<T>(
  path: string,
  fallback: T,
  options?: RequestInit & { revalidate?: number }
): Promise<T> {
  try {
    const result = await fetchStrapi<T>(path, options);
    return result.data ?? fallback;
  } catch (error) {
    console.error(`[Strapi] Failed to fetch ${path}:`, error);
    return fallback;
  }
}

    await fetchStrapi('/api/leads', {
      method: 'POST',
      body: JSON.stringify({ data: { name, email, company, message } }),
      revalidate: 0,
    });

    // 2. 企微通知
    if (process.env.WECOM_WEBHOOK_URL) {
      await fetch(process.env.WECOM_WEBHOOK_URL, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          msgtype: 'markdown',
          markdown: {
            content: `**新询盘**\n>姓名: ${name}\n>邮箱: ${email}\n>公司: ${company}\n>留言: ${message}`,
          },
        }),
      });
    }

    return NextResponse.json({ success: true });
  } catch (error) {
    console.error('[Contact API Error]', error);
    return NextResponse.json(
      { error: '提交失败，请稍后重试' },
      { status: 500 }
    );
  }
}

候选优势劣势状态

决策关键点： 确认现有开发团队（AU-03 项目等）主要使用 React 还是 Vue。

11.2 CDN 服务商

需求（来自 PRD §7.11）：

  • 国内用户走国内加速节点
  • 海外用户走境外加速节点
  • 支持按地理位置智能路由
  • ICP 备案在国内节点完成后方可上线
候选适用场景状态
阿里云 CDN + 阿里云海外节点国内覆盖好，海外节点一般⬜ 待评估
腾讯云 CDN与阿里云类似⬜ 待评估
阿里云（国内）+ Cloudflare（海外）国内外分别最优，但需双平台维护⬜ 待评估
单一全球 CDN（Cloudflare）管理简单，但国内速度受限，ICP 备案需额外配置❌ 不推荐（ICP 合规风险）

备注： 对象存储（OSS 等）选型随 CDN 服务商确定，建议同一厂商减少跨平台传输成本。
