说明

本文档为功能需求清单，供开发团队排期使用。详细业务逻辑、页面结构、CMS 模块规格见《架构设计文档》。


优先级含义目标
P0官网上线必须具备第一版上线
P1上线后第一轮迭代SSO、权限体系、语言扩展

一、前台页面需求

EP-01 首页
作为访客，我需要在首页快速判断"旭衡能解决我的问题"，并找到对应的解决方案或产品入口。

功能清单

功能点优先级说明
Hero 区块（主标语 / 副标语 / CTA）P0文案由 CMS 配置，支持多语言
三路分流 CTA（解决方案 / 开发者中心 / 联系我们）P0开发者中心跳转占位页
解决方案场景卡片列表（5个）P0自动从 CMS 解决方案内容拉取
产品概览卡片P0自动从 CMS 产品内容拉取
兼容品牌 Logo 墙P0从 CMS 兼容品牌管理拉取
信任背书数字展示P1P0 阶段硬编码，P1 迁入 CMS
底部 CTA 区块P0文案通过多语言字典配置
Hero 内容可在 CMS 后台修改并实时生效，不需要代码发布。
三路分流 CTA 点击后正确跳转，开发者中心显示「正在开发中」占位页。
EP-02 解决方案（5个场景页）
作为采购方，我需要了解旭衡在我所在场景（储能/充电/VPP/电能质量）的完整解决方案，并快速找到相关产品。

功能清单

功能点优先级说明

「本方案使用的产品」跳转卡片P0从产品-解决方案关联自动展示
应用案例区块P1从案例库关联，无案例时自动隐藏
导航菜单正确高亮当前场景P0—

产品跳转卡片点击后正确跳转至对应产品页。
